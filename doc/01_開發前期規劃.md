# 網頁動畫範例與技術解構網站 - 開發前期規劃

本文件旨在規劃一個專門展示「網頁動畫範例」並解構其「背後技術與程式碼」的網站。網站的整體視覺、互動體驗與版面配置，將參考 BananaX 的簡潔、卡片化風格進行設計。

## 1. 核心需求與目標

*   **展示用途**：收錄各種 Web 動畫效果（如 Hover 互動、進場動畫、捲動視差、3D 渲染等）。
*   **技術解構**：提供每一個動畫範例背後使用的技術堆疊（如 CSS, GSAP, Framer Motion, Three.js 等），並可查看核心程式碼片段。
*   **方便快速新增**：提供能極速擴充內容的機制。透過「輸入網頁 URL」的操作，系統可自動抓取該網站並分析該範例背後使用的技術架構。
*   **視覺風格**：採用現代化、乾淨的卡片式網格佈局（Grid/Masonry），強調內容與視覺的純粹性。

## 2. 視覺與互動設計規劃

*   **固定式標頭 (Sticky Header)**：包含全域搜尋、深淺色模式 (Dark/Light) 切換。
*   **橫向分類標籤 (Category Tabs)**：提供快速篩選標籤。
*   **響應式網格 (Responsive Grid)**：主內容區使用卡片排列。
*   **乾淨的卡片外觀**：白色或微弱的毛玻璃背景，配合細微圓角與淺色投影。
*   **互動元素 (Interactive Elements)**：卡片懸停 (Hover) 動畫。
*   **彈窗預覽 (Modal Dialog)**：點擊卡片後以彈出視窗顯示詳細內容（動畫放大預覽、AI分析出的技術介紹、程式碼片段複製）。

## 3. 系統選型與架構

*   **全端框架**：`Next.js` (App Router)
*   **樣式與排版**：`Tailwind CSS` + `Vanilla CSS`
*   **動畫主力套件**：`GSAP`, `Framer Motion`
*   **程式碼高亮展示**：`Prism.js`
*   **解析與抓取工具**：`Cheerio` / `Puppeteer` (用於抓取輸入目標網站)，整合 `Google Gemini API` (或其他 LLM) 以自動分析並摘要程式碼。
