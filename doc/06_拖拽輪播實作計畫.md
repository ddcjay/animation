# 拖拽與鍵盤驅動照片輪播實作計畫 (Siena.film 逆向工程)

本文件紀錄從 `siena.film` 網站逆向工程後，提取出的「按住滑鼠左鍵拖拽」與「鍵盤上下方向鍵」驅動照片切換特效之核心實作指引，供下一階段（或新對話）實作測試網頁時參考。

## 1. 核心技術選型
*   **前端框架**：Next.js (App Router) + React
*   **動畫引擎**：GSAP (核心 Timeline 綁定)
*   **拖拽偵測**：GSAP Observer (或 GSAP Draggable / 原生 Pointer Events)
*   **視覺特效**：CSS Transforms (處理視差與文字錯位)、可選配 WebGL (處理圖片液化)

## 2. 實作架構與機制

### A. 容器與狀態管理
*   **取消原生捲動**：外層容器設定 `overflow: hidden`，高度設為 `100vh`。
*   **全域進度變數**：維護一個 `progress` 狀態（範圍 0 到 1，或 0 到 N-1 個 Slide），代表當前整個輪播的時間軸進度。

### B. 事件攔截 (GSAP Observer)
利用 `Observer.create` 統一管理輸入：
*   **滑鼠/觸控拖拽**：在 `onDrag` 或 `onChangeY` 中，獲取垂直位移變化量 (`deltaY`)，並依照比例更新 `progress`。
*   **滑鼠釋放 (Snap)**：在 `onUp` 或 `onStop` 事件中，計算當前 `progress` 距離哪個 Slide 最近，或是根據滑動速度 (Velocity) 決定是否前往下一張，接著透過 Tween 將 `progress` 動畫過渡到整數點（吸附效果）。
*   **鍵盤輸入**：攔截 `ArrowUp` 與 `ArrowDown`，按下時直接觸發目標 Slide 的切換動畫。

### C. GSAP Timeline 視覺綁定
建立一個跨越所有 Slide 的 Master Timeline，預設為暫停 (`paused: true`)：
1.  **照片視差 (Parallax)**：在 Timeline 中定義背景圖片與前景文字不同速率的 `y` 軸位移。
2.  **文字切片 (Text Slice)**：將標題分為三層疊加，在切換過程中，給予中間層不同的 `x` 或 `y` 偏移量。
3.  **進度驅動**：每當 `progress` 更新時，直接呼叫 `timeline.progress(progress)` 或 `timeline.tweenTo()` 來即時反應視覺狀態。

## 3. 開發步驟建議

1.  **環境建置**：建立測試用頁面 (例如 `app/test-slider/page.tsx`)。
2.  **DOM 結構搭建**：實作全螢幕固定容器，內部放置 3-5 張假圖片與標題結構（包含切片文字的 DOM）。
3.  **基礎 Timeline**：撰寫 GSAP 動畫腳本，單純透過時間軸控制 Slide 1 到 Slide N 的轉場。
4.  **綁定 Observer**：加入滑鼠拖拽與鍵盤事件，將輸入轉化為 Timeline 的進度控制，並加入放開滑鼠時的自動吸附 (Snap) 邏輯。
5.  **細節打磨**：加入動態速度計算，若速度過快則加大文字錯位幅度或圖片形變。
