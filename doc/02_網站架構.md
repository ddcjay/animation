# 網站架構 (Web Architecture)

本文件紀錄「網頁動畫展演與技術解構網站」的檔案結構與技術堆疊配置方式。

## 1. 目錄結構 (Directory Structure)

專案基於 Next.js App Router 建立：

```text
/
├── doc/                 # 專案開發與規格相關文件
├── public/              # 靜態資源 (Images, Icons, Fonts)
├── src/
│   ├── app/             # Next.js App Router 頁面路由與 API
│   │   ├── api/         # 後端 API (處理 URL 抓取與 AI 分析)
│   │   ├── globals.css  # 全域 CSS 與 Tailwind 變數
│   │   ├── layout.tsx   # 全域版面與共用 Header/Footer
│   │   └── page.tsx     # 前台首頁主畫面 (網格卡片列表)
│   ├── components/      # React 獨立重複使用 UI 組件
│   │   ├── ui/          # 基礎按鈕、輸入框等
│   │   └── animation/   # AnimationCard, Modal 等核心展示組件
│   ├── lib/             # 共用工具函式 (爬蟲邏輯、LLM 串接、格式轉換)
│   └── types/           # TypeScript 型別定義
```

## 2. 核心技術模組

### 前端 (UI & Animation)
*   **React + Tailwind CSS**：處理 UI 層。
*   **GSAP / Framer Motion**：控制網頁中各卡片的進場與互動動畫。
*   **Prism.js**：在詳細 Modal 彈窗內顯示 `<code>` 高亮。

### 後端 (Data Fetching & AI Engine)
*   **Next.js API Routes**：提供 `/api/analyze` 路由供前端呼叫。
*   **Cheerio/Puppeteer**：負責模擬訪問與抓取目標 URL 的 DOM、`<script>` 及 CSS 特徵。
*   **LLM API (Gemini/OpenAI)**：將抓到的字串餵給大語言模型，分析出：「使用了哪些套件、核心實作程式碼片段」，最後回傳結構化的 JSON。
