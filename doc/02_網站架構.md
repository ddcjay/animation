# 網站架構 (Web Architecture)

本文件紀錄「網頁動畫展演與技術解構網站」的檔案結構與技術堆疊配置方式。

## 1. 目錄結構 (Directory Structure)

專案基於 Next.js App Router 建立（以下為實際存在的檔案與資料夾）：

```text
/
├── doc/                              # 專案開發與規格相關文件
│   ├── 01_開發前期規劃.md
│   ├── 02_網站架構.md               # 本文件
│   ├── 03_開發任務進度紀錄.md
│   └── 04_API規格.md
├── public/                           # 靜態資源
│   ├── file.svg
│   ├── globe.svg
│   ├── next.svg
│   ├── vercel.svg
│   └── window.svg
├── src/
│   ├── app/                          # Next.js App Router 頁面路由
│   │   ├── favicon.ico
│   │   ├── globals.css               # 全域 CSS 與 Tailwind 色彩變數
│   │   ├── layout.tsx                # Root Layout (全域版面)
│   │   └── page.tsx                  # 前台首頁 (Hero + CategoryTabs + AnimationGrid)
│   ├── components/                   # React 可重複使用 UI 組件
│   │   ├── layout/                   # 佈局相關元件
│   │   │   ├── Header.tsx            # 頂端固定導覽列 (Logo、搜尋、色彩模式切換)

│   └── data/                         # 靜態資料
│       └── mockAnimations.ts         # 模擬動畫範例資料 (目前 6 筆)
├── .env.example                      # 環境變數範本 (含 GEMINI_API_KEY 等)
```

> [!NOTE]
> `src/app/api/`、`src/lib/`、`src/types/` 尚未建立，預計在 Phase 3（AI URL 分析機制）開發時一併建立。

## 2. 核心技術模組

### 前端 (UI & Animation)
*   **React + Tailwind CSS**：處理 UI 層，使用客製化色彩 CSS 變數管理 Dark/Light 佈景。
*   **Framer Motion**：目前用於 `AnimationCard.tsx` 的卡片懸停上浮效果 (`whileHover`)。
*   **GSAP**：已安裝，預計 Phase 4 整合捲動進場動畫。
*   **Prism.js**：已安裝，預計用於 Modal 彈窗內的程式碼高亮展示。
*   **Lucide React**：介面圖示（Search、Moon、Sun、Monitor、Code2、ExternalLink 等）。

### 後端 (Data Fetching & AI Engine)
*   **Next.js API Routes**：預計建立 `/api/analyze` 路由（待 Phase 3 實作）。
*   **Cheerio**：已安裝，負責解析目標 URL 的 HTML DOM 結構與 `<script>` 引入特徵。
*   **Google Gemini API**：將抓取的原始碼送交 LLM 分析動畫技術架構，並回傳結構化 JSON（待 Phase 3 實作）。
